[# KhozaMarket](https://muhammedkazi11.github.io/KhozaMarket/index.html)


The Login Page serves as the entry point to the application and is responsible for giving users access to different sections based on their credentials. Users enter their email and password, which are checked against a predefined app_users table in the database. Depending on the role assigned to the user, the system redirects them accordingly: administrators are sent to the Admin Dashboard, while regular users are directed to the main marketplace page (index.html). This login system is intentionally simple and does not persist authentication across pages, making it suitable for development and demonstration purposes while still supporting role-based access.
The Admin Dashboard acts as the central control panel for managing the entire system. It provides an overview of the platform by displaying key statistics such as the total number of users, products, and orders stored in the database. From this page, administrators can navigate to specialized management sections including User Management, Product Management, and Order Monitoring. Its primary purpose is to give administrators a clear and efficient interface for overseeing the marketplace and performing high-level administrative tasks.
The Cart and Checkout Page is responsible for handling the purchasing process and ensuring a smooth user experience. It retrieves items stored in the browser’s local storage and displays them along with their individual prices and the total cost of the order. Users are able to remove items from the cart and proceed to checkout without needing to log in. During checkout, an order is created in the database, the selected products are stored in the order_items table, and the cart is cleared. The user is then redirected back to the homepage, completing the transaction in a simple and frictionless manner.
The Product Details Page provides users with a focused view of a selected product, displaying relevant information such as the product name, price, description, and seller details. From this page, users can quickly add the product to their cart using the “Add to Cart” button, which stores the item in local storage and redirects them directly to the cart page. This design encourages a fast and intuitive purchasing flow by minimizing unnecessary steps between product discovery and checkout.
The Sell Product Page allows products to be added to the marketplace and linked to specific users acting as sellers. It includes input fields for the product name, price, and description, along with a dropdown menu populated from the profiles table to select a seller. When a product is submitted, it is saved in the products table with a status of “pending,” ensuring that it must be reviewed before being made publicly visible. This page supports a controlled listing process and enables the system to function as a multi-vendor platform.
The Product Management Page enables administrators to review and control all products submitted to the platform. It displays each product along with its details and current status, distinguishing between pending and approved listings. Administrators can approve products to make them visible on the marketplace or delete them if necessary. This page plays a key role in maintaining the quality and integrity of the platform by ensuring only appropriate content is available to users.
The Order Monitoring Page provides administrators with a detailed view of all orders placed through the system. Each order is displayed with its total value, status, and a list of the items included in the purchase. Administrators can update the status of orders, for example by marking them as completed once processed. This page ensures that all transactions can be tracked and managed effectively, supporting operational oversight and fulfillment processes.
The User Management Page allows administrators to oversee and control user roles within the platform. It displays all users stored in the profiles table along with their email addresses and assigned roles. Administrators can promote users to admin status or revert them back to regular users as needed. This functionality ensures that access levels are properly managed and that administrative privileges are assigned appropriately.
